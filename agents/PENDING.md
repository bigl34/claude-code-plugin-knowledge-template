# Pending Corrections

This file tracks corrections and new information discovered during conversations that need user review before being applied to the knowledge base.

## Workflow

1. Agent logs corrections here during conversations
2. User reviews entries periodically
3. If approved: Update knowledge file, move entry to LEARNINGS.md
4. If rejected: Remove entry with reason

## Format

```markdown
### [YYYY-MM-DD] Brief description
**File**: knowledge/products.md (or "new file needed")
**Current**: What the knowledge base says (or "not documented")
**Proposed**: What it should say
**Source**: User correction / Discovered in conversation
**Status**: PENDING
```
